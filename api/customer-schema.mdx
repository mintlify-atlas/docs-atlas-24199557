## Overview

The `customerSchema` validates customer data from Stripe, including address details, name, email, and currency preferences.

## Schema Definition

```typescript
import { z } from "zod";

export const customerSchema = z.object({
    address: z.object({
        city: z.string().nullable(),
        country: z.string().nullable(),
        line1: z.string().nullable(),
        line2: z.string().nullable(),
        postal_code: z.string().nullable(),
        state: z.string().nullable(),
    }),
    name: z.string().nullable(),
    email: z.string().email().nullable(),
    currency: z.string().toLowerCase(),
});
```

## Inferred TypeScript Type

```typescript
type Customer = z.infer<typeof customerSchema>;
```

## Usage Examples

### Basic Validation

```typescript
import { customerSchema } from '@/lib/stripe-ui-types';

const customerData = {
  address: {
    city: "San Francisco",
    country: "US",
    line1: "123 Market St",
    line2: "Suite 400",
    postal_code: "94102",
    state: "CA"
  },
  name: "Jane Doe",
  email: "jane@example.com",
  currency: "USD"
};

const validated = customerSchema.parse(customerData);
console.log(validated.currency); // "usd" (automatically lowercased)
```

### Safe Parsing with Error Handling

```typescript
const result = customerSchema.safeParse(customerData);

if (result.success) {
  console.log("Valid customer:", result.data);
} else {
  console.error("Validation errors:", result.error.issues);
}
```

### Partial Customer Data

```typescript
const partialCustomer = {
  address: {

    line2: null,

    state: "NY"
  },
  name: null,
  email: null,
  currency: "usd"

const validated = customerSchema.parse(partialCustomer); // Valid
```

### Type Inference in Components

```typescript
import { z } from 'zod';
import { customerSchema } from '@/lib/stripe-ui-types';

type Customer = z.infer<typeof customerSchema>;

interface CustomerCardProps {
  customer: Customer;
}

export function CustomerCard({ customer }: CustomerCardProps) {
  return (
    <div>
      <h2>{customer.name ?? "Anonymous"}</h2>
      <p>{customer.email ?? "No email"}</p>
      <address>
        {customer.address.line1}<br />
        {customer.address.line2 && <>{customer.address.line2}<br /></>}
        {customer.address.city}, {customer.address.state} {customer.address.postal_code}
      </address>
    </div>
  );
}
```

## Validation Rules

- **address**: Required object with all nested fields nullable
- **name**: Optional string (can be null)
- **email**: Optional string (can be null), but must be valid email format when present
- **currency**: Required string, automatically converted to lowercase
